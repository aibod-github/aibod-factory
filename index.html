現場DXの「あるある課題」サンプル

Excel や Access、CSV、紙帳票などで回している「ちょっと困った」「誰もやりたくない」作業の例です。
AIBOD Factory は、これらをそのまま専用ツールにしてお渡しします。
SME-001

結合セルだらけのExcelが、システムに取り込めない

Excel 帳票整形

見やすくするために結合セル・色付け・空白行を多用したExcel帳票。
人は読めるが、設備やクラウドサービスには「列がずれている」「形式が合わない」と怒られる…。

現状：目視で確認してコピペで調整 対象：製造・品質・営業管理
AIBOD Factoryなら？ →
Solution
Excel → 機械が読みやすいCSV/JSONに自動整形

既存のExcel帳票のルールを読み取り、結合セル解除・列揃え・不要行削除 を自動で実行。
設備やクラウドにそのまま投入できるCSV/JSONを出力します。

効果：インポート前の手作業をほぼゼロに 形態：Windows用ツール / スクリプト
← 戻る
この課題で相談する
SME-002

毎月10個以上のExcelを「貼り替え」して集計している

Excel 月次集計

売上・仕入・稼働など、部門ごとにバラバラなExcel。
月末になると担当者が「今年分.xlsx」にコピー＆ペーストし続ける作業が発生している。

現状：担当者しかやり方がわからない 対象：管理部門 / 経営企画
AIBOD Factoryなら？ →
Solution
複数Excel → 自動集計レポート生成

指定フォルダにExcelを入れるだけで、決まった形の集計表・グラフ を自動生成。
シート名や列位置の揺れにも対応し、担当者のノウハウをツールに固定します。

効果：月3〜5時間 → 数分 形態：デスクトップアプリ or Webアプリ

SME-003

キーエンスなどの装置が mdb で出すが、中身が見られない

Access / mdb 計測器

測定結果が Microsoft Access形式（.mdb）で保存されるが、
現場PCにはAccessが入っておらず、担当者のPCだけでしか中身を見られない。

現状：担当者PCで開いてCSVに書き出し → 共有 対象：品質管理・技術
AIBOD Factoryなら？ →
Solution
mdb → テーブル別CSV自動分解ツール

mdbファイルを読み込み、テーブルごとのCSVに一括変換。
そのままExcelや他システムに読み込める形で出力します。

効果：担当者依存の解消・自動化 形態：Windowsツール（ダブルクリックで実行）

SME-004

Accessマクロで動く仕組みが、壊れても直せない

Access 既存資産

以前作ってもらったAccessマクロが、PC更新やOfficeバージョンアップで動かなくなった。
作った人も退職しており、誰も直せない。

ロジックを読み解き、Python / Web にリプレイス

Accessマクロの動きを分析し、PythonスクリプトやWebアプリとして作り直します。
将来の改修・拡張も行いやすい構成にリファクタリング。

効果：ブラックボックスの解消 形態：オンプレ / クラウド両対応

SME-005

毎月のCSVから「特定項目だけ」手で集計している

CSV 自動集計

取引先やシステムから毎月送られてくるCSV。
その中の「特定列だけ」をフィルタ・集計して、グラフや報告資料を作っている。

現状：担当者の「手順メモ」頼み 対象：営業企画・管理部門

Solution
CSV → 条件抽出 ＋ 集計 ＋ グラフを自動化

条件（例：特定仕入先・特定品目など）をツールに登録し、
フォルダにCSVを入れてボタンを押すだけで、必要な集計表とグラフを作成します。

効果：属人化の解消・スピードアップ 形態：デスクトップ or Web

画像・CSVファイルがフォルダに山積みで、整理できない

効果：探す時間と事故の大幅削減 形態：常駐スクリプト or 定期実行バッチ

Factory Services｜工場でできること

量産済みソフトだけでなく、「設計」「試作」「製造」の工程ごとにサポートします。
Design as a Service

Blueprint Studio（設計支援）

現場の課題をヒアリングし、SVA/VLP を使って構造化。 どの範囲を標準モジュールとし、どこをカスタムするかを含めた 「ソフトウェア設計図」を提供します。

Prototype as a Service

Prototype Line（試作）

設計図をもとに、まずは小さく動く試作版を製造。 画面やレポート、AIモデルの組み合わせなど、 数週間レベルで検証できるアウトプットを出荷します。

Manufacturing as a Service

Production Line（製造・量産）

試作で確認した要件をもとに、本番運用向けのソフトウェアとして製造。 テスト、自動デプロイ、監視まで含めた“工場レベルの製品”を提供します。

Step 1

困りごとヒアリング＆ツール設計

「このエクセルを毎月手で直している」「mdbからデータを出したい」など、 今やっている作業をそのまま見せてください。
それをもとに、どんなツールにすると一番ラクかを一緒に整理します。

Step 2

まずは小さく動く試作ツール

いきなり大きなシステムにはせず、まずは1つの作業が楽になる小さなツール をお出しします。
現場で試していただき、「もう少しここをこうしたい」を反映していきます。

Step 3

安定版ツールとして納品・改善

試作で使い方が固まったら、現場で長く使える安定版として作り込みます。
Windows用exe化や社内サーバーへの設置など、 お客様のIT環境に合わせてご提供します。
